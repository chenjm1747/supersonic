## 三、后端编译

### 3.1 设置环境变量

```powershell
cd E:\trae\supersonic
$env:JAVA_HOME = "D:\java\jdk-21.0.10.7-hotspot"
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
```

### 3.2 编译命令

```powershell
# 快速编译（推荐，每次代码修改后使用）
& "D:\maven-mvnd-1.0.5-windows-amd64\bin\mvnd.exe" clean package -DskipTests -pl launchers/standalone -am

# 完整编译（首次或依赖变更时使用）
& "D:\maven-mvnd-1.0.5-windows-amd64\bin\mvnd.exe" clean package -DskipTests
```

### 3.3 编译产物

| 文件 | 说明 |
|------|------|
| `launchers\standalone\target\launchers-standalone-1.0.0-SNAPSHOT.jar` | Thin JAR（约 0.1 MB），**不可直接运行** |
| `launchers\standalone\target\launchers-standalone-1.0.0-SNAPSHOT-bin.tar.gz` | 完整包（约 800+ MB），**包含所有依赖** |

## 四、前端编译

### 4.1 安装依赖（如未安装 pnpm）

```powershell
npm install -g pnpm
```

### 4.2 编译前端

```powershell
cd E:\trae\supersonic\webapp\packages\supersonic-fe
pnpm install
pnpm build:os
```

### 4.3 部署前端

```powershell
cd E:\trae\supersonic
xcopy /E /Y "webapp\packages\supersonic-fe\dist\*" "launchers\standalone\target\classes\static\"
```

## 五、启动服务

### 5.1 停止现有服务

```powershell
taskkill /F /IM java.exe
```

### 5.2 启动命令

```powershell
cd E:\trae\supersonic
$env:JAVA_HOME = "D:\java\jdk-21.0.10.7-hotspot"
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"

& "D:\java\jdk-21.0.10.7-hotspot\bin\java.exe" -Xmx2g -Xms1g -jar "E:\trae\supersonic\launchers\standalone\target\launchers-standalone-1.0.0-SNAPSHOT.jar"
```

> ⚠️ **必须解压 tar.gz 后才能用 `-jar` 方式启动**，jar 文件不包含依赖。

## 八、快速命令序列

```powershell
cd E:\trae\supersonic
$env:JAVA_HOME = "D:\java\jdk-21.0.10.7-hotspot"

```

## 九、常见问题

| 问题 | 解决方案 |
|------|---------|
| 编译失败 | 检查 `$env:JAVA_HOME` 是否指向 JDK 21 |
| 启动后立即关闭 | 检查端口占用、数据库连接、Ollama 服务是否正常 |
| API 返回 500 | 查看 `s2-error.log` 排查具体错误 |

排查命令：

```powershell
# 检查端口
netstat -ano | findstr "9080"

# 检查数据库
& "D:\Program Files\pgAdmin 4\runtime\psql.exe" "postgresql://postgres:Huilian1234@192.168.1.10:5432/postgres" -c "SELECT 1;"

# 检查 Ollama
Invoke-WebRequest -Uri "http://192.168.1.10:11435/api/tags" -Method Get

# 查看错误日志
Get-Content "E:\trae\supersonic\launchers\standalone\logs\s2-error.log" -Tail 100
```
