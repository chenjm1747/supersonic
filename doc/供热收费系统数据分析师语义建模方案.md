# 供热收费系统数据分析师 - 语义建模与指标维度设计方案

## 一、系统概述

### 1.1 系统背景
本系统是供热收费管理系统（charge_zbhx），覆盖用户管理、面积管理、计费结算、收费管理、稽查管理、客户服务等核心业务模块。

### 1.2 业务核心实体
| 实体 | 说明 | 核心表 |
|------|------|--------|
| 用户 | 热用户基本信息 | customer |
| 面积 | 用户供热面积信息 | area, sf_mj_t |
| 结算 | 面积费用结算 | sf_mjjs_t, sf_js_t |
| 收费 | 实际收费记录 | sf_mjsf_t, pay_order |
| 稽查 | 用热稽查管理 | jc_plan, jc_check_feedback |
| 停供/复供 | 供热状态变更 | sf_mjtg_t, sf_mjcw_t |
| 发票 | 发票管理 | sf_fpxx_t |

## 二、语义建模设计

### 2.1 主题域划分（Domain）

根据业务分析，将系统划分为以下5个主题域：

```
┌─────────────────────────────────────────────────────────────────┐
│                        供热收费系统主题域                           │
├─────────────┬─────────────┬─────────────┬─────────────┬───────────┤
│   用户域     │   计费域     │   收费域     │   运营域     │  质量域   │
│  (Customer) │  (Billing)  │ (Collection)│ (Operation) │(Quality) │
└─────────────┴─────────────┴─────────────┴─────────────┴───────────┘
```

### 2.2 概念模型（Conceptual Model）

#### 2.2.1 用户域实体关系
```
用户(customer)
    │
    ├──[1:N]──> 面积(area/sf_mj_t)      用户面积信息
    │    │
    │    └──[1:N]──> 结算(sf_mjjs_t)    按采暖期结算记录
    │         │
    │         ├──[1:N]──> 收费(sf_mjsf_t)    收费明细
    │         └──[1:N]──> 停供复供(sf_mjtg_t) 停供/复供记录
    │
    ├──[1:N]──> 更名过户(sf_gmgh_t)     更名过户申请
    ├──[1:N]──> 稽查(jc_plan_detail)   稽查计划明细
    └──[1:N]──> 微信绑定(wechat_bind)  微信绑定信息
```

#### 2.2.2 计费域实体关系
```
结算(sf_mjjs_t) ──核心业务单据──
    │
    ├──[归属]──> 采暖期(cnq)          2024-2025, 2025-2026
    ├──[归属]──> 费用类别(fylb)        采暖费/基本热费/计量热费
    ├──[构成]──> 应收金额(ysje) = 基础应收 + 计量费用
    ├──[状态]──> 欠费金额(qfje)        应收费 - 实收费
    └──[事件]──> 业务事件(event)       面积变更/停供/复供
```

## 三、指标体系设计（Metrics）

### 3.1 指标分类框架

```
┌──────────────────────────────────────────────────────────┐
│                      指标分类                              │
├─────────────────┬────────────────────────────────────────┤
│   原子指标       │ 不可再分的最基本指标                      │
│   派生指标       │ 在原子指标基础上加入时间/区域/业务限定      │
│   衍生指标       │ 多个指标计算得出的复合指标                  │
└─────────────────┴────────────────────────────────────────┘
```

### 3.2 核心原子指标库

#### 3.2.1 用户类指标
| 指标编码 | 指标名称 | 计算公式 | 数据来源 |
|---------|---------|---------|---------|
| USER_CNT | 用户总数 | COUNT(DISTINCT customer_id) | sf_dim_customer |
| USER_NEW_CNT | 新增用户数 | COUNT(DISTINCT customer_id) WHERE rwrq=本期 | sf_dim_customer |
| USER_BLACK_CNT | 黑名单用户数 | COUNT(*) WHERE kz_hmd=1 | sf_dim_customer |
| USER_AREA_CNT | 有面积用户数 | COUNT(DISTINCT customer_id) WHERE sf_dim_area.zf=0 | sf_dim_area |

#### 3.2.2 面积类指标
| 指标编码 | 指标名称 | 计算公式 | 数据来源 |
|---------|---------|---------|---------|
| AREA_SFMJ_SUM | 收费面积汇总 | SUM(sfmj) | sf_dim_area |
| AREA_ZDMJ_SUM | 建筑面积汇总 | SUM(zdmj) | sf_dim_area |
| AREA_TGMJ_SUM | 停供面积汇总 | SUM(tgmj) | sf_fact_stop_restart |
| AREA_ACTIVE_CNT | 在供面积数 | COUNT(*) WHERE gnzt='正常' | sf_dim_area |

#### 3.2.3 计费类指标
| 指标编码 | 指标名称 | 计算公式 | 数据来源 |
|---------|---------|---------|---------|
| BILL_YSJE_SUM | 应收金额汇总 | SUM(ysje) | sf_rpt_charge |
| BILL_JCYS_SUM | 基础应收汇总 | SUM(jcys) | sf_rpt_charge |
| BILL_QFJE_SUM | 欠费金额汇总 | SUM(qfje) | sf_rpt_charge |
| BILL_HJJE_SUM | 核减金额汇总 | SUM(hjje) | sf_rpt_charge |
| BILL_ZKJE_SUM | 折扣金额汇总 | SUM(zkje) | sf_rpt_charge |
| BILL_WYJE_SUM | 违约金额汇总 | SUM(wyje) | sf_rpt_charge |

#### 3.2.4 收费类指标
| 指标编码 | 指标名称 | 计算公式 | 数据来源 |
|---------|---------|---------|---------|
| PAY_SFJE_SUM | 实收金额汇总 | SUM(sfje) | sf_rpt_payment |
| PAY_CNT | 收费笔数 | COUNT(*) | sf_rpt_payment |
| PAY_ONLINE_CNT | 线上收费笔数 | COUNT(*) WHERE sffs IN ('微信','支付宝','网银') | sf_rpt_payment |
| PAY_OFFLINE_CNT | 线下收费笔数 | COUNT(*) WHERE sffs IN ('现金','POS','转账') | sf_rpt_payment |
| PAY_ONLINE_AMT | 线上收费金额 | SUM(sfje) WHERE sffs IN ('微信','支付宝','网银') | sf_rpt_payment |
| PAY_OFFLINE_AMT | 线下收费金额 | SUM(sfje) WHERE sffs IN ('现金','POS','转账') | sf_rpt_payment |

#### 3.2.5 收费率指标
| 指标编码 | 指标名称 | 计算公式 | 说明 |
|---------|---------|---------|------|
| PAY_RATE | 收费率 | PAY_SFJE_SUM / BILL_YSJE_SUM * 100% | 核心运营指标 |
| PAY_RATE_TG | 停供用户收费率 | 停供用户实收 / 停供用户应收 | 需特殊计算 |
| ARREARS_RATE | 欠费率 | BILL_QFJE_SUM / BILL_YSJE_SUM * 100% | 欠费风险指标 |

### 3.3 派生指标定义

#### 时间维度派生
| 派生指标 | 基础指标 | 限定条件 |
|---------|---------|---------|
| 本期应收金额 | BILL_YSJE_SUM | cnq = 当前采暖期 |
| 上期应收金额 | BILL_YSJE_SUM | cnq = 上年采暖期 |
| 本月收费金额 | PAY_SFJE_SUM | sfrq BETWEEN 月初 AND 月末 |
| 本期新增用户 | USER_NEW_CNT | rwrq BETWEEN 采暖期初 AND 当前 |

#### 区域维度派生
| 派生指标 | 基础指标 | 限定条件 |
|---------|---------|---------|
| 分公司应收 | BILL_YSJE_SUM | org_level1 = XX分公司 |
| 热力站应收 | BILL_YSJE_SUM | org_level2 = XX热力站 |
| 小区应收 | BILL_YSJE_SUM | org_level3 = XX小区 |

### 3.4 衍生指标定义

| 衍生指标 | 计算公式 | 业务含义 |
|---------|---------|---------|
| 收费完成度 | 本期实收 / 本期应收 | 反映收费进度 |
| 户均热费 | 实收总额 / 收费户数 | 用户平均热费 |
| 单位面积热费 | 实收总额 / 收费面积 | 元/平方米 |
| 欠费催缴率 | 已催缴欠费额 / 欠费总额 | 催费效果 |
| 增量收费率 | (本期实收-上期实收) / 上期实收 | 增长率 |
| 预付费覆盖率 | 预付费用户数 / 总用户数 | 信用用户占比 |

## 七、分析主题与常用查询

### 7.1 收费分析主题

#### 主题1: 收费进度分析
```sql
-- 采暖期收费进度报表
SELECT
    cnq AS 采暖期,
    SUM(ysje) AS 应收金额,
    SUM(sfje) AS 实收金额,
    SUM(qfje) AS 欠费金额,
    ROUND(SUM(sfje) / SUM(ysje) * 100, 2) AS 收费率,
    COUNT(DISTINCT customer_id) AS 用户数,
    ROUND(SUM(sfje) / COUNT(DISTINCT customer_id), 2) AS 户均热费
FROM sf_rpt_charge
WHERE zf = 0
GROUP BY cnq;
```

#### 主题2: 收费方式分析
```sql
-- 各收费方式占比分析
SELECT
    sffs AS 收费方式,
    COUNT(*) AS 收费笔数,
    SUM(sfje) AS 收费金额,
    ROUND(SUM(sfje) / (SELECT SUM(sfje) FROM sf_rpt_payment WHERE zf=0) * 100, 2) AS 金额占比
FROM sf_rpt_payment
WHERE zf = 0
GROUP BY sffs
ORDER BY 收费金额 DESC;
```

### 7.2 用户分析主题

#### 主题3: 用户结构分析
```sql
-- 用户类型分布
SELECT
    yhlx AS 用户类型,
    COUNT(*) AS 用户数,
    SUM(sfmj) AS 收费面积,
    ROUND(AVG(sfmj), 2) AS 户均面积
FROM sf_dim_area a
JOIN sf_dim_customer c ON a.customer_id = c.customer_id
WHERE a.zf = 0 AND c.zf = 0
GROUP BY yhlx;
```

### 7.3 欠费分析主题

#### 主题4: 欠费用户分析
```sql
-- 欠费用户清单
SELECT
    c.customer_code AS 用户编码,
    c.customer_name AS 用户名称,
    c.address AS 地址,
    SUM(m.ysje) AS 应收金额,
    SUM(m.sfje) AS 实收金额,
    SUM(m.qfje) AS 欠费金额
FROM sf_rpt_charge m
JOIN sf_dim_customer c ON m.customer_id = c.customer_id
WHERE m.zf = 0 AND m.qfje > 0
GROUP BY m.customer_id
ORDER BY 欠费金额 DESC;
```

### 7.4 区域分析主题

#### 主题5: 分公司收费对比
```sql
-- 分公司收费指标对比
SELECT
    org_level1 AS 分公司,
    COUNT(DISTINCT customer_id) AS 用户数,
    SUM(ysje) AS 应收金额,
    SUM(sfje) AS 实收金额,
    ROUND(SUM(sfje) / SUM(ysje) * 100, 2) AS 收费率
FROM sf_rpt_charge
WHERE cnq = '2025-2026' AND zf = 0
GROUP BY org_level1
ORDER BY 收费率 DESC;
```
